2. Diagramme de Cas d'Utilisation

Acteurs du Système

👤 SECRÉTAIRE / ADMINISTRATEUR
  • Gestion complète des patients (Ajouter, Consulter, Modifier, Supprimer)
  • Gestion complète des médecins (Ajouter, Consulter, Modifier, Supprimer)
  • Gestion des rendez-vous (Prendre, Consulter, Modifier, Annuler)
  • Accès au tableau de bord
💰 COMPTABLE
  • Créer des factures
  • Consulter les factures
  • Modifier les factures
  • Enregistrer les paiements
  • Imprimer les factures
  • Suivre les paiements
📊 DIRECTEUR
  • Consulter le tableau de bord
  • Visualiser les statistiques
  • Exporter la base de données
  • Générer des rapports

Cas d'Utilisation Principaux

CU1 : Ajouter un Patient
Acteur : Secrétaire
Description : Permet d'enregistrer un nouveau patient dans le système avec toutes ses informations personnelles et médicales.
CU2 : Prendre un Rendez-vous
Acteur : Secrétaire
Description : Permet de planifier un rendez-vous entre un patient et un médecin à une date et heure précises.
CU3 : Créer une Facture
Acteur : Comptable
Description : Permet de générer une facture pour un patient suite à une consultation ou un traitement.
CU4 : Enregistrer un Paiement
Acteur : Comptable
Description : Permet d'enregistrer un paiement partiel ou total d'une facture et de mettre à jour son statut.
CU5 : Consulter le Tableau de Bord
Acteur : Directeur, Secrétaire
Description : Affiche les statistiques globales du système et les derniers rendez-vous.

3. Règles de Gestion

RG1 - Patients : Un patient doit avoir au minimum un nom, prénom et téléphone. Un patient peut avoir plusieurs rendez-vous et plusieurs factures.
RG2 - Médecins : Un médecin doit avoir une spécialité et un numéro de licence unique. Seuls les médecins actifs peuvent recevoir de nouveaux rendez-vous.
RG3 - Rendez-vous : Un rendez-vous concerne un seul patient et un seul médecin. Les statuts possibles sont : Programmé, Confirmé, Terminé, Annulé.
RG4 - Factures : Chaque facture a un numéro unique. Le montant payé ne peut pas dépasser le montant total. Le statut est calculé automatiquement.
RG5 - Sécurité : Les suppressions de patients/médecins entraînent la suppression en cascade des rendez-vous associés. Les factures conservent les informations même si le médecin est supprimé.
